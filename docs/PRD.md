# PRD — 나라카 증권거래소 (NARAKA Stock Exchange)

> 대구 동성로 요괴 컨셉카페 "나라카"의 8월 이벤트 데이(주식데이)를 위한 모의 주식 거래 웹 서비스.
> 손님들이 약 한 달간 가상 화폐로 요괴 테마 주식을 사고팔고, 최종 순위권 4명(잠정)에게 상품을 지급한다.

- 작성일: 2026-07-11
- 이벤트 기간: **2026-08-01(토) ~ 2026-08-30(일)** — 월·화 휴장, 개장일 22일
  - ⚠️ 08-31은 월요일(휴장)이라 마지막 개장일은 **08-30(일)**. 순위도 이날 종가로 확정
- 개발 마감 목표: **2026-07-28** (오픈 전 테스트 3일 확보)

## 1. 목표

### 1.1 비즈니스 목표
- 카페 방문 유도: 가입 코드·방문 보너스를 매장에서만 발급 → 참여하려면 매장에 와야 함
- 재방문 유도: 방문 보너스(1일 1회)로 한 달간 반복 방문 동기 부여
- 커뮤니티 재미: 손님들 사이에 "오늘 뭐 샀냐" 대화거리 제공

### 1.2 제품 목표
- 캐주얼 유저도 3분 안에 이해하는 단순한 거래 시스템 (시장가 매매만)
- 한 달 내내 긴장감 유지: 뉴스 힌트, 급등락, 역전 가능한 밸런스
- 상품이 걸린 이벤트이므로 **조작 불가능한 공정성** (모든 검증 서버 사이드)

### 1.3 비목표 (하지 않는 것)
- 지정가 주문, 호가창, 공매도, 신용거래 — 구현하지 않음
- 실제 금전 거래·환전 — 절대 없음 (가상 화폐만, 상품은 오프라인 지급)
- 네이티브 앱 — 모바일 웹으로 충분

## 4. 가격 엔진 — "하루 먼저 뽑는" 사전 생성 경로 방식

시나리오를 미리 정하지 않되 뉴스 힌트를 가능하게 하는 핵심 설계.

### 4.1 일일 사이클

```
매일 22:00 장 마감 직후 (일일 배치, 1회):
1. 오늘 종가 확정, 정산 (금요일이면 배당 지급)
2. 내일 각 종목의 "일일 편향(bias)" 랜덤 추첨
   - 매일 2~4개 종목에 이벤트 배정: {+30, +20, +10, -10, -20, -30}% 중 가중 추첨
   - 나머지 종목은 중립 (등급별 기본 변동성만)
3. 내일 15:00~22:00의 84틱 가격 경로를 전부 사전 생성해 DB 저장
   - 편향을 드리프트로 갖는 랜덤워크, 직전 개장일 종가 ±30% 클램프
   - 경로 생성 시 VI(거래정지) 구간도 함께 계산해 저장
4. 편향 추첨 결과로부터 내일 자 뉴스 자동 생성
5. 오늘 실제 결과로부터 "공시" 뉴스 자동 생성
```

**휴장일 처리 (월·화 휴장)**
- 배치는 매일 22:00 실행하되 조건부로 동작:
  - 정산(종가 확정·배당)은 **당일이 개장일일 때만** 수행
  - 경로·뉴스 생성은 **익일이 개장일일 때만** 수행 — 예: 수요일 경로·뉴스는 화요일 22:00 배치가 생성하며, 기준가(±30% 클램프 기준)는 직전 개장일인 일요일 종가
- 개장일 판정: 요일 기반(월·화 휴장) + `config`로 예외 지정 가능 (임시 휴무·특별 개장 대응)
- 휴장일 UI: 장 상태 "휴장" 표시, 매매 차단(서버 검증 포함), 차트·랭킹 열람은 가능

### 4.2 장중 동작
- 현재가 = 사전 생성 경로에서 `floor(현재시각 / 5분)` 위치의 틱 값 → **장중 크론 불필요**, 서버는 읽기만 함
- 매수/매도는 항상 "현재 틱 가격"으로 서버에서 체결 (클라이언트가 보내는 가격은 신뢰하지 않음)
- 프론트엔드는 틱 사이 5분 동안 가격을 미세하게 흔들리는 것처럼 **보간 연출** (실제 체결가는 틱 값 고정)

### 4.3 변동성 완화 장치 (연출 요소)
- **점프 이벤트**: 경로 생성 시 낮은 확률(우량 0.2%/일반 0.5%/테마주 1.5% per 틱)로 한 틱에 ±2~7% 급변 주입 — 차트 재미 + VI 발동 재료
- **VI (종목별)**: 직전 틱 대비 **±6% 이상 급변** → 해당 종목 5분(1틱) 거래정지 + 화면 배너 (초안 "10분 내 ±10%"는 발동 확률 0이라 시뮬레이션 후 조정, 2026-07-12)
- **서킷브레이커 (시장 전체)**: 어드민이 수동 발동 → 전 종목 10분 거래정지, 사이트 전체 경고 연출. 발동 시 남은 당일 경로를 현재 지점부터 재생성 가능 (깜짝 폭락/폭등 이벤트용)

### 4.4 상한가/하한가
- 모든 틱 가격은 `직전 개장일 종가 × [0.7, 1.3]` 범위로 클램프
- 상한가/하한가 도달 시 해당 방향 진행 불가 (매매 자체는 가능)
- UI: 상한가 도달 종목은 빨간 배지 "上", 하한가는 파란 배지 "下" 연출

## 5. 뉴스 시스템

### 5.1 뉴스 3등급

| 등급 | 적중률 | 내용 | 발행 시점 |
|------|--------|------|----------|
| 📰 공시 | 100% (과거 사실) | 전날 실제 등락 결과 요약 | 일일 배치 시 |
| 📢 정식 뉴스 | 90% | 다음 날 편향에 대한 방향 힌트 | 일일 배치 시 |
| 💬 찌라시 | 55% | 다음 날 편향 힌트지만 45% 확률로 방향이 틀림 | 일일 배치 시 |

- 정식 뉴스의 "10% 오보"와 찌라시의 "45% 오보"는 뉴스 생성 시 확률적으로 방향을 뒤집어 만든다
- 이벤트가 배정된 종목이라고 반드시 뉴스가 나오지는 않음 (커버리지 ~70%) → 뉴스 없는 급등도 존재
- 이벤트 없는 종목에 대한 페이크 찌라시도 가끔 발행

### 5.2 뉴스 본문 생성
- 종목 × 편향 등급(+30/+20/+10/중립/-10/-20/-30)별 **한국어 템플릿 풀**을 미리 작성 (종목당 등급별 3개 이상, 요괴 세계관 플레이버)
- 일일 배치가 템플릿에서 랜덤 선택 + 변수 치환 → 운영 손 안 감
- 어드민이 수동 뉴스 작성도 가능 (특별 이벤트 공지 등)

- 다크 테마 기본 (나라카 = 지옥/요괴 무드, 증권 HTS 감성)

### 6.1 디자인 방향 — "아기자기한 지옥"

레퍼런스: `docs/design-refs/naraka-logo.png` (브랜드 로고), `docs/design-refs/character-style-ref.png` (카페 캐릭터 일러스트 스타일)

- **톤**: 무서운 지옥이 아니라 **귀여운 요괴들의 지옥**. 카페 공식 일러스트(치비 캐릭터, 굵은 외곽선, 큰 눈, 따뜻한 색감)와 결이 맞아야 한다. 호러 연출 금지, 아기자기+포근한 다크.
- **키 비주얼**: 검은 나비 날개 + 핏빛 붓글씨 로고. 로고의 먹 번짐·붓 텍스처를 헤더/스플래시 포인트로 활용.
- **팔레트 (시안)**:
  - 배경: 딥 차콜/먹색 (순수 검정보다 살짝 따뜻하게)
  - 브랜드 포인트: 로고의 핏빛 레드, 보조로 버건디/웜 레드 (레퍼런스의 커튼 배경 느낌)
  - 텍스트/서피스: 크림·한지색 계열 (차가운 흰색 지양)
  - 등락 시멘틱 컬러: 상승 빨강 / 하락 파랑 (국내 증시 관례) — **브랜드 레드와 톤을 구분**해 혼동 방지 (예: 등락 레드는 밝고 채도 높게, 브랜드 레드는 어둡고 깊게)
- **형태**: 둥근 모서리, 부드러운 그림자, 배지·이모지 적극 활용. HTS의 정보 밀도는 유지하되 딱딱한 격자 느낌은 피한다.
- **캐릭터 활용**: 빈 상태(보유 없음, 뉴스 없음), 온보딩, VI/서킷브레이커 배너 등에 요괴 캐릭터 일러스트/이모지로 아기자기함 강화.

## 7. 인증·계정

- **가입 코드**: 어드민이 일괄 생성 → 매장에서 손님에게 1인 1코드 지급, 1회 사용 후 소멸 → 다계정 억제
- **방문 보너스 코드**: 날짜별 코드 (매일 교체, 매장 게시) → 입력 시 +100,000원, 계정당 1일 1회
- 로그인: 닉네임 + 비밀번호 (이메일 불필요 — 카페 손님 진입장벽 최소화)
- 닉네임: 2~8자, 중복 불가, 랭킹에 공개되므로 욕설 필터
- 부정행위 대응: 어드민의 계정 정지·거래 롤백 기능

## 8. 어드민 페이지

- 유저 관리: 검색, 자산 조회, 계정 정지

|------|------|
| 프레임워크 | Next.js 16 (App Router) + React 19 — **SSR/API routes 사용 (정적 export 아님)** |
| 언어 | TypeScript 5 (strict) |
| 스타일링 | TailwindCSS v4 + shadcn/ui |
| 서버 상태 | TanStack Query v5 (5분 틱 폴링 + 보간) |
| 클라이언트 상태 | Zustand v5 (UI 상태만 — 게임 데이터는 전부 서버) |
| 폼/검증 | React Hook Form v7 + Zod v4 |
| 차트 | lightweight-charts (TradingView 오픈소스) 또는 recharts — Phase 4에서 결정 |
| DB/인증 | **Supabase** (Postgres + Auth + RLS) |
| 배치 | **Supabase pg_cron** — 일일 배치 1회 (22:00 마감 처리 + 익일 경로/뉴스 생성) |
| 배포 | Vercel (무료 티어) |

### 9.1 아키텍처 핵심 결정
- **장중 크론 없음**: 5분 틱은 사전 생성 경로를 읽기만 하므로 Vercel Hobby(크론 1일 1회 제한)로도 충분
- **배치 구조** (2026-07-12 확정): 경로 생성·편향 추첨 로직은 **TS 엔진**(`src/lib/engine/`, 시드 RNG로 결정적) — 몬테카를로 시뮬레이션과 운영 배치가 같은 코드를 쓴다. DB 반영은 `apply_daily_batch()` 단일 트랜잭션(멱등). 트리거는 pg_cron→pg_net HTTP 호출로 `/api/cron/daily-batch` (CRON_SECRET 인증, 등록은 T-702 배포 시)
- **모든 돈 계산은 서버(Postgres 함수/트랜잭션)에서**: 매수/매도는 단일 트랜잭션으로 잔고 검증 → 체결 → 기록. 클라이언트는 표시만
- 자산 컬럼은 정수(원 단위) — 부동소수점 금지
- **인증은 커스텀(닉네임+비밀번호), Supabase Auth 미사용**: 모든 DB 접근은 서버(service role) 경유, RLS는 전 테이블 기본 차단 — 클라이언트 직접 접근 봉쇄

### 9.2 DB 스키마 개요

```
users            — id, nickname, password_hash, cash, is_admin, is_banned, created_at
signup_codes     — code, used_by, used_at
visit_codes      — date, code
visit_claims     — user_id, date (unique)
stocks           — code, name, tier, description, listed
daily_ticks      — stock_code, date, tick_index(0~83), price, is_halted
daily_summary    — stock_code, date, open, high, low, close, bias(추첨 결과)
holdings         — user_id, stock_code, quantity, avg_price
trades           — id, user_id, stock_code, side, quantity, price, fee, created_at
news             — id, date, stock_code, grade(공시/뉴스/찌라시), title, body, published_at
config           — key, value (이벤트 기간, 수수료율 등)
```

## 10. 게임 밸런스 (1차 튜닝 완료 2026-07-12, `npm run simulate`로 재검증)

- 일일 이벤트 추첨 가중치: ±10 (40%) / ±20 (35%) / ±30 (25%), 상승:하락 = 55:45 (전체 시장 약우상향 → 존버도 유효)
- 테마주는 이벤트 배정 확률 2배 (급등락 단골)
- **편향의 확률적 실현** (핵심 밸런스 장치): 추첨된 편향(=뉴스 재료)은 30% 확률로 방향 반전, 실현 강도 20~100% 균등. 뉴스는 원래 편향 기준으로 발행 → 뉴스 적중률 명세(90%/55%)는 유지하면서 "뉴스만 따라가면 무조건 승리"를 차단
  - 도입 근거: 실현이 확정적이면 뉴스추종 전략이 중앙값 8.4배로 압도 (목표 위반)
- 등급별 틱 변동성: 우량 0.18% / 일반 0.6% / 테마주 1.4% (per 틱)
- 시뮬레이션 결과 (개장일 22일 × 1,000회, 최종 자산 중앙값/최대/원금손실율):
  - 존버(분산) 1.04배 / 2.05배 / 40% — 안전
  - 뉴스추종 1.49배 / 8.31배 / 24% — 최적 전략이지만 압도적이지 않음
  - 테마주몰빵 0.78배 / 5.25배 / 65% — 하이리스크 역전 카드
- 목표 달성: 1위 총자산 3~10배 구간 형성 (뉴스추종 상위권), 역전 가능성 유지
