# PRD — 나라카 증권거래소 (NARAKA Stock Exchange)

> 대구 동성로 요괴 컨셉카페 "나라카"의 8월 이벤트 데이(주식데이)를 위한 모의 주식 거래 웹 서비스.
> 손님들이 약 한 달간 가상 화폐로 요괴 테마 주식을 사고팔고, 최종 순위권 3~4명에게 상품을 지급한다.

- 작성일: 2026-07-11
- 이벤트 기간: **2026-08-01 ~ 2026-08-31** (약 31일)
- 개발 마감 목표: **2026-07-28** (오픈 전 테스트 3일 확보)

## 1. 목표

### 1.1 비즈니스 목표
- 카페 방문 유도: 가입 코드·방문 보너스를 매장에서만 발급 → 참여하려면 매장에 와야 함
- 재방문 유도: 방문 보너스(1일 1회)로 한 달간 반복 방문 동기 부여
- 커뮤니티 재미: 손님들 사이에 "오늘 뭐 샀냐" 대화거리 제공

### 1.2 제품 목표
- 캐주얼 유저도 3분 안에 이해하는 단순한 거래 시스템 (시장가 매매만)
- 한 달 내내 긴장감 유지: 뉴스 힌트, 급등락, 역전 가능한 밸런스
- 상품이 걸린 이벤트이므로 **조작 불가능한 공정성** (모든 검증 서버 사이드)

### 1.3 비목표 (하지 않는 것)
- 지정가 주문, 호가창, 공매도, 신용거래 — 구현하지 않음
- 실제 금전 거래·환전 — 절대 없음 (가상 화폐만, 상품은 오프라인 지급)
- 네이티브 앱 — 모바일 웹으로 충분

## 4. 가격 엔진 — "하루 먼저 뽑는" 사전 생성 경로 방식

시나리오를 미리 정하지 않되 뉴스 힌트를 가능하게 하는 핵심 설계.

### 4.1 일일 사이클

```
매일 22:00 장 마감 직후 (일일 배치, 1회):
1. 오늘 종가 확정, 정산 (금요일이면 배당 지급)
2. 내일 각 종목의 "일일 편향(bias)" 랜덤 추첨
   - 매일 2~4개 종목에 이벤트 배정: {+30, +20, +10, -10, -20, -30}% 중 가중 추첨
   - 나머지 종목은 중립 (등급별 기본 변동성만)
3. 내일 15:00~22:00의 84틱 가격 경로를 전부 사전 생성해 DB 저장
   - 편향을 드리프트로 갖는 랜덤워크, 전일 종가 ±30% 클램프
   - 경로 생성 시 VI(거래정지) 구간도 함께 계산해 저장
4. 편향 추첨 결과로부터 내일 자 뉴스 자동 생성
5. 오늘 실제 결과로부터 "공시" 뉴스 자동 생성
```

### 4.2 장중 동작
- 현재가 = 사전 생성 경로에서 `floor(현재시각 / 5분)` 위치의 틱 값 → **장중 크론 불필요**, 서버는 읽기만 함
- 매수/매도는 항상 "현재 틱 가격"으로 서버에서 체결 (클라이언트가 보내는 가격은 신뢰하지 않음)
- 프론트엔드는 틱 사이 5분 동안 가격을 미세하게 흔들리는 것처럼 **보간 연출** (실제 체결가는 틱 값 고정)

### 4.3 변동성 완화 장치 (연출 요소)
- **VI (종목별)**: 경로상 10분 내 ±10% 이상 변동 구간 → 해당 종목 5분 거래정지 + 화면 배너
- **서킷브레이커 (시장 전체)**: 어드민이 수동 발동 → 전 종목 10분 거래정지, 사이트 전체 경고 연출. 발동 시 남은 당일 경로를 현재 지점부터 재생성 가능 (깜짝 폭락/폭등 이벤트용)

### 4.4 상한가/하한가
- 모든 틱 가격은 `전일 종가 × [0.7, 1.3]` 범위로 클램프
- 상한가/하한가 도달 시 해당 방향 진행 불가 (매매 자체는 가능)
- UI: 상한가 도달 종목은 빨간 배지 "上", 하한가는 파란 배지 "下" 연출

## 5. 뉴스 시스템

### 5.1 뉴스 3등급

| 등급 | 적중률 | 내용 | 발행 시점 |
|------|--------|------|----------|
| 📰 공시 | 100% (과거 사실) | 전날 실제 등락 결과 요약 | 일일 배치 시 |
| 📢 정식 뉴스 | 90% | 다음 날 편향에 대한 방향 힌트 | 일일 배치 시 |
| 💬 찌라시 | 55% | 다음 날 편향 힌트지만 45% 확률로 방향이 틀림 | 일일 배치 시 |

- 정식 뉴스의 "10% 오보"와 찌라시의 "45% 오보"는 뉴스 생성 시 확률적으로 방향을 뒤집어 만든다
- 이벤트가 배정된 종목이라고 반드시 뉴스가 나오지는 않음 (커버리지 ~70%) → 뉴스 없는 급등도 존재
- 이벤트 없는 종목에 대한 페이크 찌라시도 가끔 발행

### 5.2 뉴스 본문 생성
- 종목 × 편향 등급(+30/+20/+10/중립/-10/-20/-30)별 **한국어 템플릿 풀**을 미리 작성 (종목당 등급별 3개 이상, 요괴 세계관 플레이버)
- 일일 배치가 템플릿에서 랜덤 선택 + 변수 치환 → 운영 손 안 감
- 어드민이 수동 뉴스 작성도 가능 (특별 이벤트 공지 등)

## 7. 인증·계정

- **가입 코드**: 어드민이 일괄 생성 → 매장에서 손님에게 1인 1코드 지급, 1회 사용 후 소멸 → 다계정 억제
- **방문 보너스 코드**: 날짜별 코드 (매일 교체, 매장 게시) → 입력 시 +100,000냥, 계정당 1일 1회
- 로그인: 닉네임 + 비밀번호 (이메일 불필요 — 카페 손님 진입장벽 최소화)
- 닉네임: 2~8자, 중복 불가, 랭킹에 공개되므로 욕설 필터
- 부정행위 대응: 어드민의 계정 정지·거래 롤백 기능

### 9.1 아키텍처 핵심 결정
- **장중 크론 없음**: 5분 틱은 사전 생성 경로를 읽기만 하므로 Vercel Hobby(크론 1일 1회 제한)로도 충분. 일일 배치는 pg_cron으로 DB 안에서 처리
- **모든 돈 계산은 서버(Postgres 함수/트랜잭션)에서**: 매수/매도는 단일 트랜잭션으로 잔고 검증 → 체결 → 기록. 클라이언트는 표시만
- 자산 컬럼은 정수(냥 단위) — 부동소수점 금지

### 9.2 DB 스키마 개요

```
users            — id, nickname, password_hash, cash, is_admin, is_banned, created_at
signup_codes     — code, used_by, used_at
visit_codes      — date, code

stocks           — code, name, tier, description, listed
```

## 10. 게임 밸런스 초안 (테스트로 조정)

- 일일 이벤트 추첨 가중치: ±10 (40%) / ±20 (35%) / ±30 (25%), 상승:하락 = 55:45 (전체 시장 약우상향 → 존버도 유효)
- 잡주는 이벤트 배정 확률 2배 (급등락 단골)
- 시뮬레이션 검증 필수: 31일 × 1,000회 몬테카를로 돌려서 "존버 vs 단타 vs 뉴스추종" 전략별 기대수익 확인 — 특정 전략이 압도적이면 파라미터 조정
- 목표: 1위 총자산이 초기 자금의 3~10배 사이에 형성, 마지막 주까지 5위권 내 역전 가능
