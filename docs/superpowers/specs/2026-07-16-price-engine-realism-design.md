- 근거 리서치: 아티팩트 "나라카 시장 vs 실제 주식시장 — 비교 리서치" §02, §07, 권고 1
- 관련 메모리: [[ou-price-engine-7-15]], [[market-hours-operating-value]]

## 1. 배경과 문제

리서치 리포트가 짚은 나라카 가격형성의 "의도적 비현실" 중 **경로 생성 층위**의 핵심 갭:

> σ가 등급 내 상수라, 나라카 차트는 실제보다 매끄럽다 (변동성 클러스터링 미반영).

현재 `randomWalk.ts`는 등급별 **상수 σ의 순수 기하 랜덤워크(GBM) + iid 점프**다.
실제 시장의 변동성은 (a) 개장·마감에 높고 정오에 낮은 U자 시간구조, (b) 험한 구간이
뭉쳐 오는 지속성(클러스터링), (c) 큰 사건 뒤 여진, (d) 조용한 날/험한 날 레짐,
(e) 개장 갭을 가진다. 나라카 차트에는 이 중 아무것도 없어 밋밋하다.

## 2. 목표

`randomWalk.ts`의 상수 σ를 **시간에 따라 살아 있는 σ**로 바꿔, 차트를 실제 시장처럼
숨 쉬게 만든다. **드리프트(방향)는 손대지 않는다.** 아키텍처(사전생성 경로·서버권위·
무한유동성)는 그대로 유지한다.

범위: 접근 **A(방향중립 변동성 구조) + B(레짐)**. 접근 C(관측 가능한 모멘텀)는 보류.

## 3. 불변식 (위반 금지)

| # | 불변식 | 이유 |
|---|--------|------|
| I1 | **방향중립** — `bias`·`DAILY_DRIFT`는 그대로. 바꾸는 건 σ의 크기·시간구조와 시작점(갭)뿐 | 관측 가능한 방향 예측을 만들지 않아야 지정가 브라켓 무손실 차익이 자멸한다 (OU 폐기와 동일 원칙, 2026-07-15 검증) |
| I2 | **결정적 재현** — 모든 난수는 기존 `rng.ts` 시드 RNG(`rng()`, `nextGaussian(rng)`)에서만 소비 | 몬테카를로 ≡ 운영 동일 엔진 |
| I3 | **하루 기대 총변동성 보존** — U-shape·클러스터링의 평균 배율 ≈ 1로 정규화 | 우승 라인(상위 컷) 급변 방지. 레짐만 의도적으로 꼬리를 두껍게 한다 |
| I4 | **상하한 ±30% 클램프·VI ±8% 마킹 로직 불변** | 규칙은 그대로 두고, 자연스러운 군집 발동만 유도 |
| I5 | `regenerateRemainingPath`(시세조정)도 동일 σ 구조를 반영 | 장중 재생성 시 시각적 이음새 제거 |
| I6 | **틱 수 보정(`scale = TICKS_PER_DAY/totalTicks`) 유지** | 장 시간이 달라져도 하루 변동성·점프 기대치 보존 (기존 동작) |

## 4. 설계 — σ를 배율의 곱으로 재정의

현재:
```
sigma = TICK_SIGMA[tier] * Math.sqrt(scale)   // 상수
```

개선 (틱 i마다):
```
sigma_i = TICK_SIGMA[tier] * Math.sqrt(scale) * intraday(i) * cluster(h_i) * regime_mult
```

`intraday`·`cluster`·`regime`의 **하루 평균 배율이 1에 수렴**하도록 설계한다(I3, 레짐 제외).

### 4.1 인트라데이 U-shape — `intraday(i, totalTicks)`

- 개장·마감 근처 σ↑(≈1.4x), 정오 σ↓(≈0.8x)의 결정적(비확률) 시간 배율.
- 형태: `t = i / (totalTicks - 1)` (0~1). `raw = 1 + A * (2t - 1)^2` 형태의 U자(양끝 높고 중앙 낮음),
  이후 구간 평균이 1이 되도록 정규화. `A`는 U 진폭(초안 0.8).
- 난수 미소비 → RNG 스트림에 영향 없음.

### 4.2 변동성 클러스터링 (GARCH-lite) — `cluster(h_i)`

- 지속성 상태변수 `h`를 AR(1) 유사로 진화:
  `h ← 1 + rho * (h_prev - 1) + eta * shock`
  - `rho` ≈ 0.9 (지속성), `eta` ≈ 0.15 (충격 감도)
  - `shock`: 별도 가우시안(`nextGaussian(rng)`) — 방향중립(σ만 스케일)
  - `h` 초기값 1.0, `[0.5, 2.5]`로 클램프
- "험한 구간이 뭉쳐서" 온다. 방향과 무관.
- 난수 소비: 틱당 가우시안 1개 추가. 몬테카를로 시드 재현에는 영향 없으나
  **기존 재현 스냅샷은 깨진다**(RNG 소비 순서 변경) → §7 참고.

### 4.3 점프 여진 (aftershock)

- 기존 점프 발생 시(`rng() < jumpProbability`) `h`에 일시 부스트 주입:
  `h += AFTERSHOCK_BOOST` (초안 0.8). AR(1)의 `rho`로 자연 감쇠하며 이어진다.
- 큰 사건 뒤 흔들림 재현. 별도 상태 불필요(클러스터링 상태변수 재사용).

### 4.4 레짐 (B) — `regime_mult`

- 하루 시작 시 등급별 확률로 저주파 레짐 추첨:
  - `calm` (배율 0.7), `normal` (1.0), `stormy` (1.6)
- 하루 안 0~1회 마르코프 전이(저주파). **σ 배율만** 바꾼다 — 방향성(불/베어 런)은
  arb 위험이라 **제외**.
- 역할 분리: **레짐 = 저주파 뭉침**, **클러스터링 = 고주파 뭉침**.
- I3의 예외: 레짐은 의도적으로 하루 총변동성을 이동시켜 순위 꼬리를 두껍게 한다
  → 몬테카를로 재튜닝 대상(§6-2).

### 4.5 개장 갭

- 현재 `open`은 `prevClose`에서 첫 틱이 움직인 값. 여기에 tick 0 명시적 갭 항 추가:
  `price *= Math.exp(GAP_SIGMA[tier] * nextGaussian(rng))` (드리프트 없음, 방향 랜덤)
  - `GAP_SIGMA` 등급별 0.3~0.8% 초안
- 상하한 클램프 안에서 적용. 오버나이트 지정가 arb는 방향 랜덤이라 안전(I1).

## 5. 배치 (파일 영향 최소)

- 전부 `src/lib/engine/randomWalk.ts` 내부.
- `generateDailyPath`와 `regenerateRemainingPath`가 공유할 **순수 헬퍼**를 추가:
  - `pickRegime(tier, rng) -> regimeMult` (하루/구간 σ 배율)
  - `intradaySigmaMult(i, totalTicks) -> number` (결정적)
  - 클러스터링 상태 `h`는 두 루프 안에서 인라인 진화(상태변수라 헬퍼보다 루프 내 관리가 자연스러움)
- 두 함수의 σ 계산부만 교체. 새 상수 블록(등급별 U진폭·`rho`/`eta`·여진 부스트·레짐 확률/배율·갭σ)을 파일 상단에 명시.
- **API·DB·서비스·타입 변경 없음.** `Tick`/`DailyPath` 인터페이스 불변.

## 6. 검증 (`npm run simulate` — 채택 게이트)

각 항목이 통과해야 채택:

1. **차익 자멸 확인** — 브라켓/뉴스추종 전략이 여전히 손실로 자멸하는가.
   방향중립이라 통과 기대, 그러나 필수 확인(I1의 실증).
2. **우승 라인 이동** — 총자산 분포·상위 4명 컷이 기존 대비 얼마나 움직이나.
   → 레짐 확률·배율(`stormy` 비중)로 재튜닝.
3. **VI 발동 빈도** — 군집화되되 "월 며칠" 수준 유지(폭증 금지).
4. **상하한 터치 빈도** — 클러스터링·레짐으로 과도하게 늘지 않는지.
5. **등급 순서 회귀** — 우량 < 일반 < 테마 변동성 순서 유지.

파라미터(§4의 초안 수치)는 이 시뮬레이션으로 확정한다.

**검증 결과 (Task 7, `--runs 500`, 튜닝 없이 1차 통과):** 지정가브라켓 4종 전부
중앙값 <1.0배·손실율 >50%(잡주4/4 0.63배/100.0%, 잡주6/6 0.69배/99.6%,
잡주8/8 0.75배/98.6%, 전종목6 0.89배/100.0%)로 차익 자멸 유지, 존버(안정주)
손실율 30.0% < 잡주몰빵 73.2%로 등급 순서 유지, 존버(분산) 중앙값 0.99배로
개선 전 1.00배 대비 ±15% 이내 — `verify-realism.ts` 재확인 PASS.

## 7. 리스크와 유의

- **RNG 소비 순서 변경** — 클러스터링·갭이 틱당/일당 난수를 추가로 소비하므로,
  같은 시드라도 기존과 다른 경로가 나온다. 기존 재현 스냅샷/골든 테스트가 있으면 갱신 필요.
- **레짐이 밸런스를 흔든다** — 유일하게 하루 총변동성을 이동시키는 요소.
  §6-2 재튜닝을 통과하지 못하면 `stormy` 배율/확률을 낮추거나 레짐을 보류한다.
- **성능** — 틱당 가우시안 1~2개 추가. 사전생성 배치라 장중 부하 없음, 무시 가능.

## 8. 안 하는 것 (YAGNI)

- 호가창·슬리피지·수급 반영·시장충격 (미시구조 층위 — 범위 밖, 아키텍처 유지)
- 관측 가능한 모멘텀/자기상관 (접근 C — arb 위험, 보류)
- 방향성 레짐(불/베어 런) — 방향중립 위반, 제외

## 9. 성공 기준

- 차트가 "숨 쉰다": 험한 구간·조용한 구간·개장 갭·여진이 눈으로 구분된다.
- §6 검증 5항목 전부 통과, 파라미터 확정.
- 아키텍처(사전생성·서버권위·무한유동성) 및 상하한·VI 규칙 무변경.
