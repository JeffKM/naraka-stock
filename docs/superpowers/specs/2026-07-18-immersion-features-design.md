## 배경 · 결론

리서치 결론 — 코어(가격엔진·공정성·소수점·뉴스 3등급·랭킹·지정가예약)는 이미 토스급.
부족한 건 ①매일 다시 오게 만드는 **습관 루프** ②손님끼리 엮이는 **소셜 레이어** ③한 달 서사.
이 문서는 그중 개장 전 개발 대상으로 **확정된 4종**의 설계다.

## 공통 제약 (위반 금지)

1. **모든 현금 보상은 `npm run simulate`에 반영** — 상품 걸린 공정 경쟁. 순위 왜곡 검증 필수.
2. **비현금 보상 우선** — 배지·타이틀·스티커는 현금가치 0. 공정성상 안전.
3. **UI 유니코드 이모지 금지** (확정 선호) — 반응·배지·스티커는 **텍스트/일러스트(이미지 자산)**로. 유저가 붙이는 캐릭터 스티커는 이미지라 이 규칙과 별개.
4. **캐논 어휘 금지** — 배지·미션·문안에 저승·천계·도깨비·염라 등 확장 어휘 파생어 금지 (세계관 캐논 준수).
5. **돈 계산은 서버 단일 트랜잭션** — 출석 보너스 지급도 Postgres 함수로, 클라이언트 값 불신.

## 데이터 전제 (기존 자산)

- `stock_comments`: `id, stock_code, user_id, content(1~200), created_at`. RLS force, user 삭제 시 cascade. 어드민 삭제(`adminDeleteComment`) 존재. → ② 소셜 확장의 기반.
- `visit_claims (user_id, date) unique`: 방문 보너스 1일 1회 기록. `claim_visit_bonus` 함수가 코드 검증 → 기록 → `config.visit_bonus` 지급. → ① 출석 스트릭 설계의 참조 패턴.
- `config` key-value(jsonb): `initial_cash=10,000,000`, `visit_bonus=1,000,000`. 출석 보너스 금액도 config로.
- 뉴스탭(`/news`)은 이미 인스타식 카드 피드(종목 필터·상단 고정). → ② 세그먼트 확장의 기반.
- 하단 탭 3개(시세/뉴스/지갑) 미니멀 유지 — 새 탭 신설 대신 기존 탭 안에서 확장.

## ① 출석 스트릭 보너스

### 요구사항
- `/portfolio` 지갑에 "출석 보너스" 버튼 → **당일 1회, 단순 접속 기준**(매매 조건 없음).
- 스트릭 단계별 현금 지급:
  - 연속 **1~2일차: 300,000원**
  - 연속 **3~6일차: 500,000원**
  - 연속 **7일차 이상: 700,000원** (유지)
- 하루 결석(전날 미수령) 시 → 다음 수령은 **1일차(30만)로 리셋**.
- 매장 **방문 보너스(100만, 코드)와 별개로 병존** — 둘 다 같은 날 받을 수 있음.

### 데이터 모델
- 신규 테이블 `attendance_claims (user_id, date, streak int, amount bigint)` — `(user_id, date)` unique로 1일 1회 강제 (`visit_claims` 패턴 재사용).
- 스트릭 계산: 직전 수령일이 `today - 1`이면 `streak + 1`, 아니면 `1`로 리셋. (당일 KST 기준 `now() at time zone 'Asia/Seoul'`.)
- 금액 계단은 `config`로 관리 (`attendance_tiers` 등) — 밸런스 조정 시 코드 배포 없이 값만 변경.

### 서버 (Postgres 함수)
- `claim_attendance_bonus(p_user_id)` returns `{cash, streak, amount}`:
  1. `attendance_claims`에 오늘 행 insert 시도 → `unique_violation`이면 `ALREADY_CLAIMED`.
  2. 직전 수령일 조회 → streak 계산.
  3. streak 단계별 금액 산출 → `users.cash += amount`, 행에 streak/amount 기록.
- API: `POST /api/attendance` (requireUser). `ApiResponse<T>` 래퍼.

### UI
- 지갑 상단에 출석 카드: 오늘 미수령이면 "출석 보너스 받기" 버튼, 수령 후엔 "N일 연속 · 다음 보상까지 M일" 진행 표시(목표 그래디언트).
- 다음 단계 금액을 미리 보여줘 상승 기대 연출.

### 밸런스 · 공정성 (시뮬 검증 완료 — 2026-07-18)

`npm run simulate -- --attendance`로 개근(무결석) 시 매일 보너스가 현금 유입되어 그날 전략대로
재투자되는 모델을 30일 × 2,000회 몬테카를로로 검증. 결과 요약(중앙값, 초기자금 대비 배수):

| | Baseline(보너스 X) | 출석 개근 반영 |
|---|---|---|
| 존버(안정주) | 10.18배 | 29.58배 |
| 섹터소문추종(최고) | 10.92배 | 31.77배 |
| 잡주몰빵(최저) | 5.85배 | 21.82배 |
| 전략 간 스프레드 | 1.87배 | **1.48배** |
| 원금손실율 | 33~100% | 대부분 0% |

**판정 (사장님 확정):**
1. ✅ **전략 순위는 안 뒤집힘** — 오히려 전략 간 격차가 압축(1.87→1.48배). "어떤 전략이 유리한가"의 공정성은 유지.
2. ⚠️ **출석 성실도가 순위의 지배 변수** — 개근(≈29배) vs 완전 미출석(≈10배) ≈ **3배 격차**. "평범한 전략 + 개근"이 "최고 전략 + 결석"을 이김. → 이 대회는 매매 실력보다 **출석 성실도로 순위가 갈림**. 이는 카페 방문·재방문 유도라는 이벤트 본질과 부합하는 **의도된 설계**로 확정.
3. ✅ **원금손실 방어 강화** — 하위권 이탈(학습된 무기력) 방지에 긍정적.

금액 **30만/50만/70만 확정**. 스티커/배지 등 비현금은 별도로 실력·다양성 인정 보조.

## ② 소셜/토론 확장

기존 종목 댓글(`stock_comments`)을 확장. **새 기능이 아니라 확장**이며, 원본 데이터는 `stock_comments` 하나로 유지해 대화 파편화를 원천 차단한다.

### 2-1. 댓글 좋아요 (엄지업만)
- 신규 테이블 `comment_reactions (comment_id, user_id)` unique — 1인 1회 토글.
- **엄지업만** (싫어요 없음 — 소규모 지인 커뮤니티라 다운보트는 감정 상함).
- 댓글 목록 응답에 `likeCount`, `likedByMe` 포함.

### 2-2. 뉴스탭 ⟨뉴스 | 토론⟩ 세그먼트
- `/news` 상단에 세그먼트 토글 신설:
  - **뉴스**: 기존 피드(공시·정식·찌라시) 그대로 + 반응(2-3).
  - **토론**: 전 종목 댓글을 시간순으로 합친 모음 뷰. 각 댓글에 **종목 태그** → 누르면 해당 종목 상세로 이동.
- 토론 뷰는 `stock_comments`를 `created_at desc`로 전 종목 조회(작성은 여전히 종목 상세에서). 읽기 위주 집약 뷰.
- 하단 탭은 3개 유지(새 탭 없음).

### 2-3. 찌라시 반응 (믿는다/못믿는다)
- 뉴스 카드 중 **찌라시 등급**에 "믿는다 / 못믿는다" 반응 버튼(엄지업·다운) + 카운트.
- 신규 테이블 `news_reactions (news_id, user_id, kind)` — kind = up/down, `(news_id, user_id)` unique.
- 55% 적중률이라 반응이 갈리는 게 재미. (일반 뉴스/공시 반응 여부는 구현 시 결정 — 기본은 찌라시만.)

### 2-4. 스티커
- 유저가 댓글에 **제작진 큐레이션 스티커**를 붙임 (자유 이미지 업로드 X — 모더레이션 폭탄 방지).
- 스티커 세트: **세계관 캐릭터(옥자·미호·멜·바나·펫) 짤 + 기타 웃긴 스티커**. 초기 세트 이미지 자산은 별도 제작.
- 데이터: `stock_comments`에 `sticker_id text null` 추가하거나, 댓글 본문에 스티커 참조 토큰. (구현 시 택1 — 별도 첨부 컬럼 권장.)
- 스티커 카탈로그는 정적 자산 + 메타(id, 파일, 라벨)로 관리.

### 운영 · 모더레이션
- 기존 어드민 댓글 삭제(`adminDeleteComment`)·계정 정지(`is_banned`) 재사용. 신고 기능은 범위 밖(어드민 직접 삭제로 커버).

## ③ 성취 배지·타이틀

### 요구사항
- **현금 가치 0** — 순수 명예. 랭킹·프로필·댓글 작성자 옆에 노출.
- 수익률 외 다양한 플레이를 인정해 하위권 이탈 방지.

### 배지 카테고리 (초기 세트 예시)
- **플레이 스타일**: 존버의 화신(장기 보유), 단타 요괴(거래 횟수), 잡주 도박사(테마주 비중), 뉴스 사냥꾼(찌라시 적중).
- **오너 팬덤**: 옥자/미호/멜/바나 계열주 집중 보유 → "옥자 가문의 집사" 등.
- **순간 포착**: 첫 수익 실현, 상한가 매도, VI 직전 탈출.
- **습관**: 출석 스트릭 7일/14일/30일 달성 → ①과 자산 공유.

### 데이터 · 판정
- `badges` (정의: id, 이름, 설명, 카테고리, 아이콘 자산) + `user_badges (user_id, badge_id, earned_at)`.
- 판정 시점: 대부분 **일일 배치**에서 집계(보유·거래·스트릭 기반)로 부여. 순간 포착형은 거래 트랜잭션/이벤트 시 부여.
- 아이콘 일러스트는 별도 제작 — **미제작 시 텍스트/심볼로 임시 대체**(개발과 디커플).

### UI
- 프로필/지갑에 획득 배지 그리드. 랭킹 목록·댓글 작성자 옆 대표 배지 1개.

## ④ 경량 온보딩

풀 인터랙티브 튜토리얼은 과함(YAGNI). **상황별 툴팁 + 첫 행동 보상**만.

### 요구사항
- **상황별 툴팁**: 상하한 배지·VI 배너·서킷브레이커 등에 "?" → 짧은 설명 팝오버. `/guide`의 정적 규칙을 필요한 지점에 분산.
- **첫 매수 축하**: 유저의 첫 매수 체결 시 축하 연출 + (가능하면) 배지 부여 → 유능감 즉시 형성.
- 첫 매수 여부는 유저의 거래 존재로 판정(별도 플래그 불필요, 또는 `users`에 `onboarded_at` 정도).

### UI
- 툴팁은 shadcn `Popover`/`Tooltip` 재사용. 첫 매수 축하는 체결 성공 응답을 트리거로 토스트/모달.

## 구현 순서

1. **① 출석 스트릭** — 밸런스 영향 크므로 먼저. `simulate` 재검증 병행. 독립적이라 빠름.

3. **③ 배지·타이틀** — 스트릭·스티커와 자산 공유. 판정은 배치 편입.
4. **④ 온보딩** — 다른 기능이 있어야 툴팁 붙일 대상이 생김 → 마지막.

**병렬(디자인 리소스):** 스티커 짤 + 배지 일러스트는 개발과 별개로 제작 착수해야 7/28에 맞음. 미제작 시 텍스트/심볼 임시 대체.
