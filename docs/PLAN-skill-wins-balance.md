# 실력자 우승 밸런스 — 실제 반영 계획

> 목표: **"단타 아닌, 정보를 읽는 실력자가 최종 총자산 상위 4명(=상품)에 들 확률 1위"**.
> 근거·검증: 밸런스 하네스(`scripts/simulate.ts`) 몬테카를로 + 크라운 토너먼트. 풀틱(4,320) 재검증 완료(2026-07-20).
> 관련 메모리: skill-wins-balance-harness.

## 0. 검증된 결론 (요약)

- **진단**: 단일 총자산 크라운은 평균이 아니라 "오른쪽 꼬리(최대값)"를 보상 → 변동성 몰빵이 이김. 시장이 ~10배 우상향이라 변동성=순수 상방. 현행에선 **랜덤 단타조차 크라운 42%**.
- **해법(풀틱 검증)**: 도박 꼬리를 죽이고(잡주 하방 + 매도수수료) 스킬 신호를 키우면(섹터강도 + 종목뉴스 채널), 실력 전략(교차검증)이 상위4의 ~69%를 차지, 블라인드·도박은 전멸.
- **핵심 원리**: 스킬 = 톤/소문을 "읽는" 게 아니라 **초반 시세 브레이크로 확인(교차검증)**. 그래서 방향을 애매하게 둘수록 게으른 블라인드만 죽고 확인하는 실력자는 멀쩡(톤 단독 51% → 교차검증 진입 81%).

## 확정 파라미터 (2026-07-20, config A)

Phase 0 완화 스윕 결과 채택. 검증된 가혹값(drift −1.5·점프30·수수료2%)이 잡주 손실 99.7%로 과해, 실력 리드는 유지하면서 잡주 복권성을 살리는 완화값으로 확정.

| 레버 | 파일/위치 | 현행 → 확정 |
|---|---|---|
| 섹터강도 `SECTOR_MAGNITUDE` | `src/lib/engine/bias.ts` | 15 → **25** |
| 잡주 드리프트 `DAILY_DRIFT.wild` | `src/lib/engine/randomWalk.ts` | −0.2 → **−1.0** |
| 잡주 점프상승확률 `JUMP_UP_PROBABILITY.wild` | `src/lib/engine/randomWalk.ts` | 0.5 → **0.35**(35:65) |
| 매도수수료 `sell_fee_bp` | `config` (마이그레이션) | 50 → **150**(0.5%→1.5%) |

검증(288틱·필드72명): 종목뉴스교차검증 상위4 42.6%·소문교차검증 26.5%(실력 69%), 단타 상위4 16.7%(=기준선), 잡주몰빵 상위4 7.1%·손실 95.7%·최대 2.9배(복권성 보존). σ 위계(0.005/0.009/0.015)는 불변 → 등급 차별화 유지.

## 롤아웃 원칙

1. **별도 브랜치**에서 진행(main 직접 수정 금지).
2. **단계별 위험×노력 순**: 기계적 "실력 우위" 수정을 먼저(작고 저위험), 콘텐츠 큰 종목뉴스 채널은 후속.
3. **섹터 소문 교차검증 채널은 이미 존재** → Phase 1–2만으로도 실력자가 상위4 리드(하네스 config D 기준 소문교차검증 상위4 39%). 종목뉴스(Phase 3)는 "두 번째 스킬 채널 + 피드 활기" 부가.
4. **env 오버라이드 패턴 유지**: 엔진 상수는 `SIM_*` env 미설정 시 운영값. 반영은 이 "운영 기본값"을 바꾸는 것.
5. 이벤트 미개장(8/1)이라 엔진 파라미터는 **다음 배치 사전생성분부터** 반영 — 라이브 충격 없음.

## Phase 3 — 종목뉴스 채널 (엔진 + 콘텐츠, 대작업)

두 번째 스킬 채널 + 중반 피드 활기. 위험·노력 최고.

- [ ] **선행 조사(필수)**: `src/lib/news/generate.ts`의 기존 **`EARLY_SIGNAL_ACCURACY` / `excludeCodes` / 조기 방향뉴스** 스캐폴딩 파악 → 신규 구현이 아니라 이걸 **재활용/확장**. (지금 비활성/부분구현 여부 확인)
- [ ] 엔진/배치: 종목 단위 초반 뉴스 생성 — `|개별 bias|≥20` 진짜 + 방향성0 필러(진짜:노이즈 1:1), **0~40% 창 분산 발행**, **방향 미표기(톤만, 정확도 ~0.6)**. 반전 노이즈는 기존 `realizeBias` FLIP_PROBABILITY 0.3 그대로
- [ ] 타이밍 재분배: `tailNewsTick`(강제 후반) → **각 종목 움직임 "직후"**(선취 0)로 확인 뉴스 분산 + **거래량/색깔 뉴스**(경로 파생)로 중반 피드 채우기 → 초반 몰림/중반 공백 해소
- [ ] **헤드페이크(펌프-덤프 함정) + 거래량 단서** (2026-07-20 하네스 검증): 종목뉴스의 일부(진짜당 **~0.3**)를 "초반에 확 튀어 순진한 가격확인을 통과 → 종가엔 꺼지는" 함정으로. **단서 = 거래량**(진짜=거래량 실림, 함정=대개 얇음, **정확도 ~0.8=불완전**). 스킬이 "초반 오르는 거 보고 사기"(1-factor)에서 **"거래량 실린 진짜 상승 vs 세력 펌프 함정 종합판단"(다층)**으로 깊어짐. 검증: 순진 추종 적중 45%·최종 0.20×(박살), 거래량 실력 적중 70%·상위4 41%. **함정률 0.5는 과함(채널 net-손해→단타 득세), 0.3이 스위트스팟.** 브라켓 exploit 안 열림(펌프-덤프 예측불가).
  - ⚠️ **다채널종합(섹터+종목뉴스 교차대조)은 드롭**: 가격 교차검증이 이미 내리는 멤버를 걸러 추가가치 없음(하네스서 소문교차검증과 동률). 각 채널은 각자 스킬이지만 둘의 "충돌 해소"는 별도 스킬 아님.
- [ ] 콘텐츠: `src/lib/news/templates.ts`에 **애매한 톤 헤드라인**(방향 안 드러나게) 라이브러리. 세계관 캐논 준수(naraka-lore-canon), 사장님 검수 필요
- [ ] UI: 뉴스 피드가 속보를 방향 라벨 없이 표시 + **거래량 표시**(단서), 하루 종일 밀도 유지 확인
- [ ] 검증: 하네스(완료) + 로컬 배치 후 **피드 시간분포 확인** + 실앱 verify(verify 스킬)

**결정 필요 ③**: Phase 3를 이번에 포함 vs Phase 1–2만 먼저 ship(섹터 채널로 충분). 권장 = 1–2 먼저 ship 후 3은 후속.
