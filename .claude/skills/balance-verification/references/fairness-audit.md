# 공정성 감사·경계면 교차검증 참조

fairness-auditor가 로드한다. 공정성 불변식 감사 + 세 검증관 결과의 경계면 교차검증(팬인 종합)을 수행한다.

## 공정성 불변식

1. **초기 자금 1,000만원 고정** — 모든 참가자 동일. (시뮬 상수 `INITIAL_CASH = 10_000_000`)
2. **방문 보너스** — 매장 방문 +보너스, 1일 1회, 날짜별 코드. 코드 재사용·중복 수령이 불가한가.
3. **출석 스트릭 보너스** — 1~2일 30만 / 3~6일 50만 / 7일+ 70만(개근 가정). 정산 로직이 스트릭을 조작으로 부풀릴 수 없는가.
4. **정수(원) 불변식** — 모든 자산·보너스가 정수인가.
5. **가입** — 매장 발급 1회용 코드 + 닉네임 + 비밀번호. 코드 1회성이 강제되는가.

## 뉴스 편향 등급

- 공시(100%)/정식뉴스(90%)/찌라시(55%) 신뢰도. 일일 배치가 익일 편향 추첨으로 자동 생성.
- 정식뉴스는 후반부(tail) 노출 — 뉴스추종 이득 제거. middle 노출 금지.
- 특정 참가자에게 정보 우위(조기 노출·해석 우위)가 가지 않는가.

## 상대 밸런스

- 42종/18섹터 로스터. 중앙값 대비 상위/하위 배수로 공정성 판단(절대 수익률 아님).
- 상위 4명(잠정) 상품 → 상위권이 운보다 전략에 의해 갈리는지 시뮬 분포로 확인.

## 경계면 교차검증 (핵심 — 개별 정상 ≠ 통합 정상)

각 검증관이 "정상"이라 해도 연결 지점에서 계약이 어긋날 수 있다. 다음을 **교차 비교**한다:

| 경계면 | 확인 사항 |
|--------|----------|
| 엔진 밴드 ↔ 거래 체결 밴드 | `generateDailyPath`의 상하한 계산과 `execute_trade`의 체결 밴드 해석이 동일한가 |
| 배치 틱 인덱스 ↔ 거래 현재가 인덱스 | `apply_daily_batch`가 채운 인덱스와 거래가 읽는 "현재 시각 틱 인덱스"가 일치하는가 |
| 배치 뉴스 편향 ↔ 엔진 realizeBias | 배치가 추첨한 편향과 엔진이 가격에 반영하는 편향이 같은 값·부호인가 |
| 지정가 settle ↔ 배치 순서 | `settle_limit_orders`와 폐장 배치의 실행 순서가 체결가에 유리·불리를 만들지 않는가 |
| 시뮬 상수 ↔ 실제 마이그/config | `simulate.ts`의 상수(수수료·초기자금·드리프트)가 실제 DB·config와 일치하는가 (시뮬이 맞아도 운영이 다르면 무의미) |

> 교차검증은 "존재 확인"이 아니라 "shape·규칙·부호가 일치하는가"다. 정적 리뷰·빌드 통과로는 못 잡는다.

## 종합 리포트 형식

`docs/reports/balance-audit-{YYYY-MM-DD}.md` (경로는 오케스트레이터 지정):

```markdown
# 밸런스·공정성 감사 리포트 — {날짜}
## 총평 (공정성 판정: 통과 / 조건부 / 위험)
## 도메인별 요약 (엔진 / 거래 / 배치)
## 경계면 교차검증 결과
## 이슈 목록 (심각도순: 공정성 위협 > 리그레션 > 개선)
| 심각도 | 이슈 | 영향 | 재현 | 담당 도메인 |
## 상충·미수집 항목 (출처 병기)
```

## 원칙

- 상충하는 검증관 결론은 삭제하지 않고 조건·출처를 병기한다.
- 리포트 일부가 누락되면 "해당 영역 미수집"을 명시하고 나머지로 진행한다.
